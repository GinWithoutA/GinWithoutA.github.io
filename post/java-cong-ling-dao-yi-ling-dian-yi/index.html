6、计算机的存储规则

在计算机中只有三种类型的数据：文本、图片、声音
不同进制在代码中的表现形式：

  • 二进制：代码中以0b开头
  • 八进制：代码中以0开头
  • 十六进制：代码中以0x开头
    十进制转化为其他进制：除基取余法（余数倒着拼接）
    汉字存储表：GB2312编码——GBK编码——Unicode编码（万国码表）

9、原码、反码、补码

1个bit只能存储0或者1，8个bit是一个字节，他是计算机的最小存储单元
原码负数的计算会有问题，例如-1+1利用原码表示结果为-2
反码的出现是为了解决原码的负数计算问题，负数的计算需要将原码转为反码之后计算，但是若运算时结果跨0会产生一个误差，因为存在+0以及-0
补码的出现为了解决反码中存在的两个0，补码会对反码+1，从而错开，屏蔽两个0的现象，程序中的负数都用补码进行存储

从而可以解释不同数据类型之间的转换，隐式转换就是数据前面补0，强制转换就是去掉前面多余的0，剩下的数据就是转换后的

13、String

1、创建String对象的两种方式
（1）直接赋值
（2）new

2、String在内存中的存储方式
（1）直接赋值：
直接赋值的String会存放在StringTable串池中，存放在堆中，若创建的String变量中的值在串池中存在，则赋值串池中的已存在字符串的地址
（2）new
动态开辟空间，不存放于串池中
3、JDK8之前的字符串拼接的底层原理
（1）若拼接的对象全部为字符，则在编译的时候就会将最终结果保存并存入串池，若串池中有，则直接赋值串池地址

    /*
     * 在这过程中至少存在
    */
    String s = "ab" + "cd" + "ef";

（2）若拼接的对象为变量加字符，则每次拼接会生成有一个StringBuilder对象，最终底层帮我们转为String对象

    /*
     * 一个加号就创建两个对象
    */
    String s = "ab";
    String ss = s + "cd";

4、JDK8之后的字符串拼接的底层原理
（1）若拼接的对象全部为字符，不变
（2）若拼接的对象为变量加字符，则拼接的时候会预估字符串的长度，并开辟数组进行存放，最终转为String对象

    /*
     * 预估4个单位
    */
    String s = "ab";
    String ss = s + "cd";

14、StringBuilder

相较于String，可以把StringBuilder看作是一个容器，内容可以变化主要用于提高字符串的操作效率
默认会创建长度为16的字节数组，若存储的容量超过，则会扩容（原容量*2+2），若还超过就直接开辟一个实际长度的数组

15、StringJoiner

可以看作是有格式控制的StringBuilder，可以指定拼接的样式

    /*
     * 第一个参数指定每个元素之间的间隔符号
     * 第二个第三个参数指定开头和结尾符号
    */
    StringJoiner sj = new StringJoiner(',', '[', ']');

16、静态变量

静态变量优先于对象出现，随着类的字节码文件进入方法区，类中的静态变量就会在堆内存中产生
静态方法只能访问静态变量和静态方法
非静态方法可以访问静态变量和静态方法，也可以访问非静态的成员变量和非静态的成员方法
静态方法中没有this关键字
